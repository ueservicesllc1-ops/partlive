# Private Chat System - PartyLiveApp

This document outlines the architecture, data models, API endpoints, rate limiting, moderation, and verification procedures for the 1-to-1 private chat system in PartyLiveApp.

## 1. Overview
The private chat system allows users to message each other in secure, real-time channels. To prevent spam and unsolicited contact, message requests are automatically created for users who are not friends (mutual followers) or if privacy settings dictate it.

## 2. Firestore Data Model
We use three main Firestore paths:

### `privateConversations/{conversationId}`
Stores metadata about the chat. The `conversationId` is generated by sorting the two participant user IDs alphabetically: `{userAId}_{userBId}` (where UserAId < UserBId).
- `participantIds`: `string[]` (Contains both user IDs)
- `participantAId`: `string`
- `participantBId`: `string`
- `status`: `'active' | 'pending' | 'rejected' | 'blocked' | 'archived'`
- `requestStatus`: `'pending' | 'accepted' | 'rejected' | 'expired'`
- `unreadCounts`: `{ [userId: string]: number }`
- `lastMessageText`: `string`
- `lastMessageSenderId`: `string`
- `lastMessageAt`: `timestamp`

### `privateConversations/{conversationId}/messages/{messageId}`
A subcollection storing the actual messages.
- `senderId`: `string`
- `receiverId`: `string`
- `type`: `'text' | 'emoji' | 'system'`
- `text`: `string` (if type is text)
- `emoji`: `string` (if type is emoji)
- `status`: `'sent' | 'delivered' | 'read' | 'deleted' | 'hidden'`
- `deletedFor`: `string[]` (user IDs who soft-deleted the message)
- `hiddenByAdmin`: `boolean` (moderation flag)

### `messageRequests/{conversationId}`
Stores pending message invitations.
- `fromUserId`: `string`
- `toUserId`: `string`
- `status`: `'pending' | 'accepted' | 'rejected'`
- `messagePreview`: `string`

## 3. Privacy & Spam Prevention
- **Friends Only**: If target user allows messages only from `friends`, messages are blocked unless they are mutual followers.
- **Followers Only**: If target user allows messages only from `followers`, messages are blocked unless the sender follows them.
- **Everyone**: Direct chat if they are friends/followers, otherwise a `pending` request is created.
- **None**: Private messages are completely blocked.
- **Blocks**: If a block exists between two users in the `blocks` collection, sending messages is prohibited.

## 4. API Endpoints

### Mobile APIs
- `GET /api/private-chat/conversations`: Fetch conversations.
- `POST /api/private-chat/messages/send`: Send a text or emoji message.
- `POST /api/private-chat/conversations/:conversationId/read`: Reset unread count.
- `POST /api/private-chat/conversations/:conversationId/accept`: Accept request.
- `POST /api/private-chat/conversations/:conversationId/reject`: Reject request.
- `POST /api/private-chat/messages/:messageId/delete-for-me`: Hide message for caller.
- `POST /api/private-chat/messages/:messageId/report`: Report message.

### Admin APIs
- `GET /api/admin/private-chat/summary`: Overall chat statistics.
- `GET /api/admin/private-chat/reported-message/:messageId/context`: Fetch 5 messages before and 5 messages after for context.
- `POST /api/admin/private-chat/messages/:messageId/hide`: Hide reported message.

## 5. Verification Checklist
To verify the private chat features:

1. **Send Message**: Go to a public profile and tap **Mensaje**. Send a message.
2. **Accept Request**: Log in as target user. Go to messages list -> tap **Solicitudes** -> Tap **Aceptar**.
3. **Mute/Archive**: Tap options (⋮) inside private chat -> tap **Silenciar** or **Archivar**.
4. **Block User**: Tap options (⋮) -> tap **Bloquear usuario**.
5. **Admin Moderation**: Report a message from mobile. Search the message ID in Next.js admin `/private-chat` page, check surrounding context, and click **Ocultar Mensaje**.
